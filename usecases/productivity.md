# Terminal and Productivity

| # | As a ... | I would like to ... | So that I can ... |
|---|----------|---------------------|-------------------|
| P1 | **Developer / power user** | open an interactive REPL session on a laptop or Chromebook that connects to the server | use any nearby device as a terminal into the central server |
| P2 | **Power user** | have multiple REPL sessions on one device or access one session from multiple devices | work flexibly across screens without losing context |
| P3 | **Developer / power user** | ask the LLM questions, generate code, and have it propose Terminals commands on my behalf, with read-only calls auto-executed and every mutating command gated on my explicit approval | get AI assistance with contextual awareness of my running system without leaving the REPL and without losing control over what runs |
| P4 | **Power user** | pick my LLM provider (OpenRouter or Ollama) and model with a sticky selection that survives detach/reattach | control cost, latency, and privacy of AI-assisted commands |
